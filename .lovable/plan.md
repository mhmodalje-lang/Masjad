

# أوقات صلاة تلقائية حسب المسجد المختار

## المشكلة الحالية
النظام الحالي يتطلب من المستخدم إدخال أوقات الصلاة يدوياً لكل يوم. المطلوب: المستخدم يختار مسجد فقط، والأوقات تتحدث تلقائياً يومياً من Aladhan API بناءً على إحداثيات المسجد.

## الحل

### 1. إعادة بناء صفحة MosquePrayerTimes.tsx
- المستخدم يختار مسجد من القائمة القريبة (OpenStreetMap)
- فور الاختيار → يتم جلب أوقات الصلاة تلقائياً من `api.aladhan.com` باستخدام إحداثيات المسجد + طريقة الحساب المناسبة للبلد
- إزالة الإدخال اليدوي للأوقات بالكامل — كل شيء تلقائي
- حفظ المسجد المختار في `localStorage` (للزوار) و `user_selected_mosque` (للمسجلين)
- عند فتح الصفحة مرة أخرى → تحميل المسجد المحفوظ تلقائياً وجلب أوقات اليوم

### 2. إنشاء hook جديد: `useMosquePrayerTimes`
- يقبل إحداثيات المسجد + طريقة الحساب
- يستخدم نفس منطق `usePrayerTimes` الحالي (Aladhan API)
- يُرجع أوقات الصلاة + وقت الجمعة (الظهر يوم الجمعة)
- يتحدث تلقائياً كل يوم (أو عند تغيير المسجد)

### 3. ربط مع صفحة PrayerTimes الرئيسية
- إذا المستخدم اختار مسجد → عرض اسم المسجد في صفحة أوقات الصلاة الرئيسية
- الأوقات المعروضة تكون حسب إحداثيات المسجد (أدق من موقع المستخدم)

### 4. التدفق النهائي
1. المستخدم يفتح "أوقات المساجد" → تظهر قائمة المساجد القريبة تلقائياً
2. يختار مسجد ← يتم حفظه فوراً
3. تظهر أوقات الصلاة تلقائياً (من Aladhan API حسب إحداثيات المسجد)
4. كل يوم جديد → الأوقات تتحدث تلقائياً بدون أي تدخل
5. يمكنه تغيير المسجد في أي وقت

### الملفات المعدّلة
- `src/pages/MosquePrayerTimes.tsx` — إعادة بناء: إزالة الإدخال اليدوي، جلب الأوقات تلقائياً من API
- `src/pages/PrayerTimes.tsx` — عرض اسم المسجد المختار إن وجد
- لا حاجة لتعديل قاعدة البيانات — الجداول الحالية كافية، فقط نستخدم `user_selected_mosque` + `mosques`
